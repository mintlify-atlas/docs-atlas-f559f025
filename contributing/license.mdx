## What GPLv3 means in plain language

GPLv3 is a **copyleft** license. The key points for users and contributors:

- **Free to use** — run KillerPDF for any purpose, personal or professional, at no cost.
- **Free to study** — the full source code is available. You can read it, learn from it, and build on it.
- **Free to modify** — change whatever you like in your own copy.
- **Free to redistribute** — share copies of the original or your modified version, with or without charge.
- **Copyleft applies on distribution** — if you distribute KillerPDF or a modified version of it, you must provide the corresponding source code and license your version under GPLv3. You cannot strip the license or add restrictions that prevent recipients from exercising these same rights.

<Note>
  The copyleft requirement is triggered by **distribution**, not by private use. You can modify KillerPDF for internal use without publishing your changes. As soon as you distribute the modified binary to others — including inside a commercial product — you must make the source available under GPLv3. Commercial rebrands that avoid open-sourcing are not permitted.
</Note>

## GPL3 source bundle

Every build of KillerPDF automatically produces a `KillerPDF-<version>-src.zip` file alongside the binary as part of the `BundleSource` MSBuild post-publish target. This ZIP contains all tracked source files (as reported by `git ls-files`) and satisfies the GPLv3 requirement to provide a corresponding source distribution with every binary release. You can find the source ZIP for each release on the [GitHub Releases page](https://github.com/SteveTheKiller/KillerPDF).
